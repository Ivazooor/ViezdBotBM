# Бот выездных фотоотчётов — настройка и запуск

Сотрудник создаёт отчёт прямо в Telegram, переписываясь с ботом: выбирает тип, проходит чек-лист,
вводит проект/дату/ответственного/комментарий, присылает фото и видео — бот собирает всё и пересылает
в рабочий чат.

## Почему файлы до 2 ГБ работают без своего сервера

Сотрудник присылает видео **боту со своего телефона** — файл загружает его приложение Telegram (лимит
2 ГБ, с Premium 4 ГБ). Бот не скачивает и не загружает файл заново, а **пересылает его по `file_id`**
(`copyMessage`). Поэтому лимит бота в 50 МБ не действует, и не нужны ни `api_id`, ни свой Bot API сервер.

## Что нужно настроить (3 переменные)

### 1. `TELEGRAM_BOT_TOKEN` — токен бота
@BotFather → `/revoke` (старый токен утекал в историю) → скопировать **новый** токен.

### 2. `TELEGRAM_CHAT_ID` — куда слать отчёты
1. Добавьте бота в рабочую группу (и дайте право писать).
2. Откройте в браузере: `https://api.telegram.org/bot<ТОКЕН>/getUpdates`
3. Напишите любое сообщение в группе и обновите страницу — найдите `"chat":{"id":-100…}`.
   Для групп id **отрицательный** (например `-1003953057200`). Это и есть `TELEGRAM_CHAT_ID`.

### 3. `ALLOWED_USER_IDS` — кто может слать отчёты
Список Telegram ID сотрудников через запятую, например `12345678,98765432`.
- Узнать ID просто: пусть сотрудник напишет боту `/start` — бот ответит «Ваш Telegram ID: …».
- Добавьте эти ID в переменную и перезапустите сервис.

> Пока список пуст — бот **никого не пускает** (защита по умолчанию).

## Деплой на Timeweb Cloud (VPS, Ubuntu)

Бот работает по long polling — домен, SSL и webhook НЕ нужны, он сам подключается к Telegram.

1. Создать Cloud Server: Ubuntu 24.04, минимальный тариф (1 CPU / 1 ГБ). Запомнить IP и пароль root.
2. Подключиться к серверу (веб-консоль Timeweb в панели или `ssh root@<IP>`).
3. Установить Node.js и git:
   ```
   curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
   apt-get install -y nodejs git
   ```
4. Скачать код и зависимости:
   ```
   git clone https://github.com/Ivazooor/ViezdBotBM.git
   cd ViezdBotBM && npm install
   ```
5. Создать файл `.env` (`nano .env`) и вписать (см. `.env.example`):
   `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`, `ALLOWED_USER_IDS`.
6. Автозапуск через pm2 (рестарт при сбое и перезагрузке сервера):
   ```
   npm install -g pm2
   pm2 start backend/server.js --name viezdbot
   pm2 save && pm2 startup
   ```
7. Проверка: `pm2 logs viezdbot` → «Запуск Telegram-бота (long polling)». Затем боту `/start`.

> Если репозиторий приватный и `git clone` просит логин — проще временно сделать репозиторий
> публичным (в коде секретов нет, токен только в `.env`) или загрузить файлы на сервер вручную.

## Как пользуется сотрудник

1. Открывает бота в Telegram → `/start`.
2. Выбирает тип отчёта → читает чек-лист → «Продолжить».
3. Вводит проект, дату, ответственного, комментарий (или «Пропустить»).
4. Присылает все фото и видео (по одному или альбомом).
5. Жмёт «📤 Отправить отчёт» — всё уходит в рабочий чат. `/cancel` — отмена.

## Локальный запуск (для проверки)

```bash
npm install
cp .env.example .env   # вписать токен, chat_id, свой ID
npm start
```
