# ViezdBot БМ — настройка на хостинге RU-CENTER (PHP, webhook)

Бот работает на обычном виртуальном хостинге (PHP) через **webhook**: Telegram сам присылает
сообщения на скрипт `bot.php`. Отдельный сервер, Node.js и `api_id` не нужны.

Файлы до 2 ГБ: сотрудник присылает фото/видео боту со своего телефона, а бот пересылает их
в рабочий чат по `file_id` — без скачивания, поэтому лимит бота 50 МБ не действует.

---

## Шаг 1. Токен бота
@BotFather → `/revoke` (старый токен утекал) → скопировать **новый** токен.

## Шаг 2. Создать config.php
Скопируйте `config.sample.php` в `config.php` и впишите:
- `token` — новый токен из шага 1;
- `chat_id` — уже вписан `-1003953057200` (чат «Фото отчёты — отдел Сервисного обслуживания»);
- `allowed_ids` — уже вписаны 12 сотрудников (добавлять/убирать — здесь);
- `webhook_secret` — придумайте случайную строку (например 20–30 букв/цифр), запомните её.

> `config.php` НЕ кладите в git — токен должен лежать только на сервере.

## Шаг 3. Залить файлы на RU-CENTER
Через **Файловый менеджер** (или FTP) панели RU-CENTER положите в папку одного из ваших
сайтов (с HTTPS), например в подпапку `viezdbot/`:
- `bot.php`
- `config.php`

Папка `data/` создастся автоматически (для состояния диалогов). Убедитесь, что у каталога
есть права на запись (обычно так и есть).

Проверка: откройте `https://<ваш-домен>/viezdbot/bot.php` в браузере — должна быть
страница «ViezdBot работает».

## Шаг 4. Включить бота в рабочем чате
Добавьте `@bmhpolabot` в группу «Фото отчёты — отдел Сервисного обслуживания»
с правом писать сообщения.

## Шаг 5. Привязать webhook
Один раз откройте в браузере (подставив токен, домен и секрет из шага 2):

```
https://api.telegram.org/bot<ТОКЕН>/setWebhook?url=https://<домен>/viezdbot/bot.php&secret_token=<СЕКРЕТ>
```

Должно вернуться `{"ok":true,"result":true,...}`. Проверить можно так:
`https://api.telegram.org/bot<ТОКЕН>/getWebhookInfo`

## Шаг 6. Проверка
Сотрудник из списка пишет боту `/start` → проходит шаги → отправляет отчёт →
файлы появляются в рабочем чате.

---

## Частые правки
- **Добавить/убрать сотрудника** — список `allowed_ids` в `config.php` (узнать ID: сотрудник
  пишет боту, бот ответит «Ваш Telegram ID: …»).
- **Сменить рабочий чат** — `chat_id` в `config.php`.
- **Отключить бота** — `https://api.telegram.org/bot<ТОКЕН>/deleteWebhook`.

## Если что-то не так
- Бот не отвечает → проверьте `getWebhookInfo` (поле `last_error_message`); часто это неверный
  URL, отсутствие HTTPS или несовпадение `secret_token`.
- «Не удалось отправить в рабочий чат» → бот не добавлен в группу или не может писать.
- Сотрудник видит «Нет доступа» → его ID не в `allowed_ids` (бот покажет ID — добавьте его).
